How Sorter Thinks

Silent, forward-only, and paranoid about safety.

1

A new file appears

Sorter uses macOS's kqueue system — the same low-level event API Finder uses — to detect file arrivals instantly. There is no timer. There is no polling. CPU usage at idle is exactly 0%.

2

Extension-first classification

The filename is checked for compound extensions first (.tar.gz, .d.ts), then single extensions. The entire classification map lives on-device. No file name or path is ever sent anywhere.

3

Safety checks, then move

Before touching anything, Sorter verifies the destination is not a system directory, the file isn't still being written (partial downloads are held for 1 second), and the source wasn't already in the right place. Only then does the file move — using macOS's own NSFileCoordinator API.

4

Forward-only, always

Sorter only acts on new arrivals. It never touches files that existed before you added a watch folder, never re-sorts on app launch, and never moves a file twice. Your existing organisation stays exactly as you left it.

Questions

Everything you need to know.

Why forward-only? Will Sorter reorganise my existing files?

No. Sorter only acts on files that arrive after it is running. It never scans or rearranges files that already exist in a folder. This is a deliberate design decision, not a limitation. Retroactive sorting is the fastest way to accidentally destroy a project structure you spent weeks building. Sorter respects your past work and only shapes what comes next.

Is it safe? What if it moves a file I needed somewhere else?

Sorter includes multiple safety layers. PathGuard checks every destination against a list of system-reserved directory names before any move happens. Files that match in-progress download extensions are held until the download is complete. If a file would land in the same folder it's already in, Sorter skips it. Conflicts are resolved by appending _1, _2, etc. — nothing is ever overwritten or deleted. Every move is logged in the Activity Log, accessible from the menu bar.

What folders can Sorter watch?

Any folder you grant access to via the standard macOS file picker. Sorter has built-in, silent permission to watch ~/Downloads (via a macOS entitlement). For all other locations — Desktop, Documents, a project folder on an external drive — you add them in Settings. Sorter stores those permissions as security-scoped bookmarks, so access survives reboots without asking again. You can remove a folder at any time and Sorter forgets it instantly.

Can I customise where specific file types go?

Yes. In Power Mode (Settings → Categories), you can reassign any individual file extension to a different category — for example, routing .py files to Code instead of Scripts, or moving .ai files to a dedicated Design folder. You can also rename any category folder, disable entire categories, or toggle Simple Mode to just switch top-level groups on and off without thinking about sub-categories.

Does Sorter run at login? Does it affect battery life?

Launch at login is opt-in and toggled explicitly in Settings. It is never enabled without your permission. When no files are arriving, Sorter consumes 0% CPU — it uses kernel-level change notifications (kqueue), not timers or polling loops. The menu bar icon is the only persistent footprint. On Apple Silicon, the power draw at idle is unmeasurable.

What's the difference between the .dmg and the Mac App Store version?

The features are identical. The .dmg is Apple Notarized — Gatekeeper will verify it before it opens, and macOS will prompt you the first time just as it does for any Notarized app. The Mac App Store version is distributed and updated through Apple's infrastructure, which some users prefer for trust or fleet-management reasons. Both are sandboxed. Choose whichever you're comfortable with.

Privacy Policy

Data Collection

Sorter does not collect, store, or transmit any personal data. All file operations occur exclusively on your local Mac. No usage analytics, crash reports, or telemetry are sent anywhere.

Permissions

Sorter requests access only to the folders you explicitly configure. These permissions are managed entirely through macOS's standard security model and can be revoked at any time in System Settings → Privacy & Security → Files and Folders.

Third-Party Services

Sorter does not integrate with any third-party service, SDK, or advertising network. There are no in-app purchases that require account creation.
